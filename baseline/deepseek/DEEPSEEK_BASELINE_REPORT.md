## 核心发现

1. **Verilog 项目**：DeepSeek V4 Flash (OpenCode) 得分 77-82%，反超官方 DeepSeek V3.2 (Codex) 的 60-71%。OpenCode agent 在 Verilog 项目上表现优于 Codex CLI。

2. **Chisel 项目**：得分仅 24-25%，远低于官方 V3.2 (Codex) 的 46-50%。两种不同 agent（OpenCode 和 OpenHands SDK）在相同模型下结果完全一致，证明瓶颈不在 agent 实现，而在 Chisel 构建体系——Scala/Chisel 编译需数分钟，agent run 期间无法获取仿真反馈，修复依赖静态分析。

3. **全部 0 基础设施错误**，所有失败均为 patch 质量问题。
